About

Building the Infrastructure
Behind the Product

I'm a System Administrator at eClerx Services with a proven track record of designing and operating enterprise-grade Linux environments. Over the past year, I've extended that foundation into full AWS cloud infrastructure — engineering custom VPCs, automating deployments with Terraform and Ansible, containerizing applications with Docker, and building GitHub Actions CI/CD pipelines from scratch.

My approach to infrastructure is grounded in real operational experience: resolving SELinux misconfigurations, hardening MariaDB post-installation, rotating exposed credentials, and debugging K3s RAM exhaustion on constrained instances. Every system I build is designed to be observable, maintainable, and cost-aware.

I hold a B.Sc. in IT from Karpagam Academy, a CISSP from (ISC)², and an Oracle Certified Expert — Database SQL. My long-term goal is a Solutions Architect role — building systems that are production-resilient at scale.

Server Health Monitor

A real-time server health monitoring dashboard built on AWS, reading live metrics directly from the Linux /proc filesystem. CPU, memory, disk I/O, and uptime surface into a Flask REST API-backed dashboard — with CloudWatch Agent integration and SNS alerting on threshold breaches.

  • Reads from /proc/stat, /proc/meminfo, /proc/diskstats
  • Flask REST API serves metrics to a responsive front-end dashboard
  • CloudWatch Agent ships custom metrics; SNS triggers on threshold breach
  • Deployed inside a custom VPC with public/private subnets across two AZs
  • HTTPS via Apache mod_ssl; session-authenticated admin interface
